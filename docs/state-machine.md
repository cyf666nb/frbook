# 状态机流转设计

## 一、租赁订单状态机

### 状态定义
| 状态码 | 状态名称 | 说明 |
|--------|----------|------|
| 0 | 待支付 | 租赁申请已提交,等待支付押金+首期租金 |
| 1 | 已支付待确认取书 | 已支付,等待出租人确认对方取书 |
| 2 | 租赁中 | 出租人确认取书,租赁计时开始 |
| 3 | 待验收 | 承租人已归还,等待出租人验收 |
| 4 | 已完成 | 验收通过,押金退还,租金结算完成 |
| 5 | 已取消 | 订单取消(支付超时/双方协商) |
| 6 | 逾期中 | 超过租赁结束时间未归还 |

### 状态流转图

```
                    ┌─────────────────────────────────────────┐
                    │                                         │
                    ▼                                         │
┌──────────┐    ┌──────────┐    ┌──────────────┐    ┌────────┴───┐
│  待支付  │───▶│已支付待  │───▶│   租赁中     │───▶│   逾期中   │
│  (0)     │    │确认取书  │    │    (2)       │    │    (6)     │
└────┬─────┘    │  (1)     │    └──────┬───────┘    └──────┬─────┘
     │          └────┬─────┘           │                   │
     │               │                 │                   │
     │ 支付超时      │取消             │归还               │归还+逾期费
     │               │                 ▼                   │
     │               │          ┌──────────┐              │
     │               │          │  待验收  │◀─────────────┘
     │               │          │   (3)    │
     │               │          └────┬─────┘
     │               │               │
     ▼               ▼               │验收通过
┌──────────┐  ┌──────────┐          │
│  已取消  │  │  已取消  │          ▼
│  (5)     │  │  (5)     │    ┌──────────┐
└──────────┘  └──────────┘    │  已完成  │
                              │   (4)    │
                              └──────────┘
```

### 状态流转触发条件

| 当前状态 | 目标状态 | 触发条件 | 操作说明 |
|----------|----------|----------|----------|
| - | 0 | 用户提交租赁申请 | 创建订单,锁定图书库存 |
| 0 | 1 | 用户完成支付 | 支付押金+首期租金到担保账户 |
| 0 | 5 | 支付超时(15分钟) | 自动取消,释放库存锁 |
| 0 | 5 | 用户主动取消 | 释放库存锁 |
| 1 | 2 | 出租人确认取书 | 开始计时,设置租期倒计时 |
| 1 | 5 | 双方协商取消 | 退还押金+租金,释放库存 |
| 2 | 3 | 承租人归还 | 等待验收 |
| 2 | 6 | 租期到期未归还 | 开始计算逾期费用 |
| 6 | 3 | 承租人归还 | 计算逾期费用 |
| 3 | 4 | 出租人验收通过 | 结算租金,退还押金余款 |
| 4 | - | 完成 | 双方互评,更新信用分 |

### 异常处理
- **逾期处理**: 每日自动从押金扣除逾期费用(日租金×1.5倍)
- **押金不足**: 逾期费用超过押金时,扣减用户信用分
- **争议处理**: 管理员介入,可强制变更状态

## 二、出售订单状态机

### 状态定义
| 状态码 | 状态名称 | 说明 |
|--------|----------|------|
| 0 | 待支付 | 买家拍下,等待付款 |
| 1 | 已支付待发货 | 已付款到担保账户,等待卖家发货 |
| 2 | 已发货待收货 | 卖家已发货,等待买家确认收货 |
| 3 | 已完成 | 交易完成,货款结算给卖家 |
| 4 | 已取消 | 订单取消(支付超时/退款) |

### 状态流转图

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  待支付  │───▶│已支付待  │───▶│已发货待  │───▶│  已完成  │
│  (0)     │    │  发货    │    │  收货    │    │   (3)    │
└────┬─────┘    │  (1)     │    │  (2)     │    └──────────┘
     │          └────┬─────┘    └────┬─────┘
     │               │               │
     │ 支付超时      │取消           │7天自动确认
     │               │               │
     ▼               ▼               │
┌──────────┐  ┌──────────┐          │
│  已取消  │  │  已取消  │◀─────────┘
│  (4)     │  │  (4)     │  退款
└──────────┘  └──────────┘
```

### 状态流转触发条件

| 当前状态 | 目标状态 | 触发条件 | 操作说明 |
|----------|----------|----------|----------|
| - | 0 | 买家拍下图书 | 创建订单,锁定库存 |
| 0 | 1 | 买家付款 | 支付到担保账户 |
| 0 | 4 | 支付超时(15分钟) | 自动取消,释放库存 |
| 0 | 4 | 买家取消 | 释放库存 |
| 1 | 2 | 卖家发货 | 上传快递单号或确认线下交付 |
| 1 | 4 | 卖家取消(退款) | 退还货款给买家 |
| 2 | 3 | 买家确认收货 | 结算货款给卖家(扣除平台抽成) |
| 2 | 3 | 7天自动确认 | 自动结算货款 |
| 2 | 4 | 买家申请退款 | 协商退款流程 |
| 3 | - | 完成 | 双方互评,更新信用分 |

### 自动结算机制
- 发货后设置7天自动结算延迟任务
- 买家确认收货时立即结算
- 结算时扣除平台抽成(默认5%)

## 三、赠送记录状态机

### 状态定义
| 状态码 | 状态名称 | 说明 |
|--------|----------|------|
| 0 | 待确认 | 接收人申请领取,等待赠送人确认 |
| 1 | 已确认待交付 | 赠送人确认,等待线下交付 |
| 2 | 已完成 | 交付完成,双方互评 |
| 3 | 已取消 | 取消赠送 |

### 状态流转图

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│  待确认  │───▶│已确认待  │───▶│  已完成  │
│  (0)     │    │  交付    │    │   (2)    │
└────┬─────┘    │  (1)     │    └──────────┘
     │          └────┬─────┘
     │               │
     │ 超时/拒绝     │取消
     │               │
     ▼               ▼
┌──────────┐  ┌──────────┐
│  已取消  │  │  已取消  │
│  (3)     │  │  (3)     │
└──────────┘  └──────────┘
```

### 状态流转触发条件

| 当前状态 | 目标状态 | 触发条件 | 操作说明 |
|----------|----------|----------|----------|
| - | 0 | 用户申请领取 | 创建赠送记录,锁定图书 |
| 0 | 1 | 赠送人确认 | 等待线下交付 |
| 0 | 3 | 赠送人拒绝 | 释放图书,其他用户可申请 |
| 0 | 3 | 24小时无响应 | 自动取消 |
| 1 | 2 | 确认交付 | 完成赠送 |
| 1 | 3 | 双方协商取消 | 释放图书 |
| 2 | - | 完成 | 双方互评 |

## 四、图书状态流转

### 状态定义
| 状态码 | 状态名称 | 说明 |
|--------|----------|------|
| 0 | 下架 | 图书已下架 |
| 1 | 上架 | 图书正常上架,可交易 |
| 2 | 已交易 | 图书已完成交易 |

### 状态流转

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│   上架   │───▶│   下架   │    │  已交易  │
│   (1)    │    │   (0)    │    │   (2)    │
└────┬─────┘    └────┬─────┘    └──────────┘
     │               │
     │ 重新上架      │
     └───────────────┘
```

### 状态变更规则
- 发布时默认状态为 1(上架)
- 用户可主动下架,状态变为 0
- 出售/赠送完成后,状态变为 2
- 租赁归还后,状态恢复为 1(可再次出租)
- 租赁中时,状态保持 1 但被锁定

## 五、用户信用分规则

### 加分规则
| 行为 | 分数变化 | 说明 |
|------|----------|------|
| 完成租赁(出租方) | +2 | 按时完成租赁交易 |
| 完成租赁(承租方) | +2 | 按时归还无逾期 |
| 完成出售(卖家) | +2 | 顺利完成出售 |
| 完成出售(买家) | +1 | 及时确认收货 |
| 完成赠送(赠送方) | +3 | 爱心赠送 |
| 完成赠送(接收方) | +1 | 完成接收 |
| 获得好评 | +1 | 每次好评 |

### 扣分规则
| 行为 | 分数变化 | 说明 |
|------|----------|------|
| 逾期归还 | -2 | 每逾期一天 |
| 取消订单(无理由) | -3 | 已支付后取消 |
| 收到差评 | -2 | 每次差评 |
| 押金不足以抵扣逾期费 | -5 | 严重违约 |
| 恶意行为 | -10 | 管理员判定 |

### 信用等级
| 分数范围 | 等级 | 权益 |
|----------|------|------|
| 90-100 | 优秀 | 押金减免10% |
| 70-89 | 良好 | 正常使用 |
| 50-69 | 一般 | 押金增加20% |
| 0-49 | 较差 | 需额外担保 |
